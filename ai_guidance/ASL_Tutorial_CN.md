## 宏观介绍

本项目将带领你完整体验一个真实的 AI 视觉识别系统——从数据准备、模型训练，到实时摄像头推断。项目的目标是：**识别美国手语（American Sign Language，简称 ASL）中的 26 个字母手势**，并通过摄像头实时显示识别结果。

整个系统由三个模块组成：

| 模块 | 文件 | 功能 |
|------|------|------|
| 数据准备 | `prepare_dataset.py` | 用 MediaPipe 提取手势关键点，生成 CSV 描述文件 |
| 模型训练 | `train_model.py` | 读取 CSV，用 PyTorch 训练全连接神经网络 |
| 实时应用 | `realtime_inference.py` | 载入训练好的模型，通过摄像头实时识别手势 |

**为什么这套架构值得学习？**

传统的图像分类 AI 需要对原始图片做大量训练，计算量庞大。本项目采用了一个关键的"中间层"技术：**MediaPipe**。它能将每张手势图片"翻译"成 21 个手部关键点的三维坐标（共 126 个数值），大幅压缩了输入维度。最终，神经网络只需要学习这些轻量级的关键点数据，而非数百万像素的图片，训练速度和精度都得到了极大提升。

这也是你在实际 AI 工程中会频繁遇到的思路：**用特征工程降低问题复杂度，而不是用更大的模型暴力拟合原始数据。**

## 第一部分：准备数据集

### 1.1 数据集来源

本项目使用 Kaggle 上公开的 ASL 字母图像数据集：

> **数据集地址：**  
> https://www.kaggle.com/datasets/debashishsau/aslamerican-sign-language-aplhabet-dataset?resource=download-directory

数据集包含 26 个英文字母手势的图片，按文件夹分类（A、B、C…Z），每个类别包含大量图片。

### 1.2 下载与目录结构

下载并解压后，将训练数据集放置到以下路径（需与代码中的路径一致）：

```
project_root/
├── ASL_Alphabet_Dataset/
│   └── asl_alphabet_train/
│       ├── A/
│       │   ├── A1.jpg
│       │   ├── A2.jpg
│       │   └── ...
│       ├── B/
│       └── ...（共 26 个文件夹）
├── 1-Prepare_dataset_V4.1.py
├── 2-Train_NN_v4.1.py
└── 3-NN_realtime_test_V4.1.py
```

### 1.3 数据采样策略

代码中设置了采样比例（`sampling_ratio = 0.05`），即每个类别只随机抽取 5% 的图片。  
这样设计的原因是：原始数据集非常大，全量处理耗时过长，5% 的采样已经足够训练出高精度模型。  
学生可以尝试调整这个参数（例如改为 `0.1` 或 `0.2`），观察对训练结果的影响。

### Prompt 1：数据准备模块

**目的：** 指引 AI 编写脚本，将原始图片批量处理为结构化的手势关键点 CSV 文件。

---

```
# [CONTEXT]
I am building an ASL (American Sign Language) alphabet recognition system.
The dataset is from Kaggle: folders named A–Z, each containing hundreds of hand gesture images.

# [GOAL]
Write a Python script that:
1. Iterates through each class folder (A–Z) in the dataset directory.
2. For each image, uses MediaPipe Hands to detect hand landmarks.
3. Normalizes all 21 landmarks by subtracting the position of landmark[0] (the wrist),
   so that the data is translation-invariant.
4. Supports up to 2 hands; if fewer are detected, pads the remaining data with zeros.
5. Adds a horizontally flipped version of each valid sample (mirror augmentation).
6. Randomly samples only `sampling_ratio` (e.g. 5%) of images per class to keep the
   dataset manageable.
7. Uses Python multiprocessing to parallelize processing across class folders.
8. Writes all results to a CSV file with columns:
   [Image Path, Class Index, Hand1_Landmark0_x, Hand1_Landmark0_y, Hand1_Landmark0_z, ..., Hand2_Landmark20_z]

# [WHY MEDIAPIPE]
MediaPipe reduces the raw image (200×200 pixels = 40,000 values) to just 126 floating-point
numbers (2 hands × 21 landmarks × 3 coordinates). This collapses the input space
dramatically, making the downstream neural network far simpler and faster to train.

# [WHY A CSV FILE]
Decoupling feature extraction from model training is good engineering practice.
The CSV can be reused for different models without re-running the expensive MediaPipe pass.

# [CONSTRAINTS]
- Use static_image_mode=True and min_detection_confidence=0.9 for MediaPipe Hands.
- Skip images where no landmarks are detected (save them to a temp folder for inspection if desired).
- Use the folder name as the class label; map A→0, B→1, …, Z→25.
- Use tqdm to show progress.
```

### Prompt 2：模型训练模块

**目的：** 指引 AI 读取 CSV 并训练一个全连接神经网络分类器。

---

```
# [CONTEXT]
I have a CSV file (`dataset_info.csv`) generated by MediaPipe hand landmark extraction.
Each row: [image_path, class_index (0–25), 126 landmark features (floats)]

# [GOAL]
Write a PyTorch training script that:
1. Reads the CSV, extracts feature columns and class labels.
2. Splits data into 80% train / 20% test (random_state=42).
3. Defines a 3-layer fully connected network:
   Input(126) → Linear(256) + BatchNorm + Dropout(0.3) → ReLU
   → Linear(128) + BatchNorm + Dropout(0.3) → ReLU
   → Linear(26) [output logits]
4. Trains with:
   - Loss: CrossEntropyLoss
   - Optimizer: Adam (lr=0.001)
   - LR scheduler: ReduceLROnPlateau (monitor val accuracy, patience=10, factor=0.1)
   - Gradient accumulation: every 4 steps
   - Early stopping: stop if val accuracy does not improve for 50 epochs
5. Saves the best model (highest val accuracy) to `best_model.pth`.
6. Saves the final model to `last_model.pth`.
7. Prints epoch loss, val accuracy, and training time per epoch.

# [WHY THIS ARCHITECTURE]
The input is only 126 numbers, so a simple 3-layer MLP is sufficient.
BatchNorm stabilizes training; Dropout prevents overfitting on the small dataset.
ReduceLROnPlateau + early stopping prevents wasted compute.

# [CONSTRAINTS]
- Device: CPU (no GPU required for this model size).
- Gradient accumulation allows effective batch size scaling without extra memory.
- Max 5000 epochs, but early stopping should trigger well before that.
```

## 第四部分：使用提示词指南 ——如何观察 AI 编程并做好质量把控

使用 GitHub Copilot（或其他 AI 编程工具）时，以下几点值得特别注意：

### 4.1 如何输入提示词

在 VS Code 中使用 GitHub Copilot Chat（侧边栏）：
1. 新建一个空的 `.py` 文件。
2. 打开 Copilot Chat（快捷键 `Ctrl+Alt+I` / `Cmd+Alt+I`）。
3. 将上方的提示词粘贴到对话框，点击发送。
4. Copilot 会生成代码草稿；点击"Insert into Editor"插入代码。
5. 对不满意的部分，可以追问：`"Modify the landmark extraction to also handle the case where only 1 hand is detected."`

### 4.2 观察 AI 的输出——需要检查的关键点

| 检查项 | 说明 |
|--------|------|
| **归一化逻辑是否一致** | 数据准备和实时推断的关键点处理方式**必须完全相同**，否则模型输入分布会偏移，导致识别失效。 |
| **CSV 列名与模型输入对齐** | 确认训练时读取的特征列（`iloc[:, 2:]`）数量等于模型输入层大小（126）。 |
| **多进程保护** | 数据准备脚本必须在 `if __name__ == '__main__':` 块内启动多进程，否则在 Windows 上会崩溃。 |
| **设备一致性** | 训练和推断都设置为 `device = torch.device("cpu")`，无需 GPU，确保代码跨平台运行。 |
| **摄像头索引** | 脚本使用 `cv2.VideoCapture(1)`（索引 1，外部摄像头）；若只有内置摄像头，改为 `0`。 |

### 4.3 常见 AI 生成错误及修正方式

**问题 1：AI 忘记 padding（补零到 126）**  
- 现象：当只检测到 1 只手时，特征向量只有 63 个值，导致模型报错。  
- 修正提示：`"Add a while loop after landmark extraction to pad the list with zeros until its length reaches 126."`

**问题 2：AI 使用绝对坐标而不归一化**  
- 现象：模型精度很低，因为坐标值随手的位置变化。  
- 修正提示：`"Subtract landmark[0]'s x, y, z from all landmarks before appending to the list."`

**问题 3：AI 忘记 `model.eval()` 模式**  
- 现象：推断时 BatchNorm/Dropout 仍在训练模式，导致预测不稳定。  
- 修正提示：`"Call model.eval() before inference, and wrap it with torch.no_grad() to disable gradient computation."`

### 4.4 调试与观察技巧

- **打印中间变量**：训练时每 10 个 epoch 打印一次 val accuracy，方便观察收敛曲线。
- **保存最佳模型**：代码已自动保存 `best_model.pth`，训练结束后用它做推断，不要用 `last_model.pth`。
- **调整采样比例**：将 `sampling_ratio` 从 0.05 调高到 0.1，观察数据量加倍对准确率的影响（通常会提升 2–5%）。
- **观察早停时机**：如果早停在第 100 个 epoch 之前触发，说明模型可能欠拟合，可以尝试增大网络宽度。

## 第五部分：架构设计的优点

### 5.1 MediaPipe 作为特征提取器——关键的降维设计

| 对比项 | 直接训练图片 | MediaPipe 关键点方案 |
|--------|-------------|----------------------|
| 输入维度 | ~40,000（200×200 图片） | 126（关键点坐标） |
| 模型参数量 | 数百万 | 约 6 万 |
| 训练时间 | 数小时（需 GPU） | 数分钟（CPU 即可） |
| 数据需求 | 数万张图片 | 数百张即可 |
| 泛化能力 | 对背景/光线敏感 | 对背景无关 |

### 5.2 归一化设计——平移不变性

将所有关键点减去腕部（landmark[0]）的坐标，使得手的位置在画面中任何地方都能被一致识别。这是一个简单但极其有效的特征工程技巧。

### 5.3 镜像数据增强——低成本扩充数据

对每张有效图片额外生成一个 x 坐标取反的版本，等价于"左右手镜像"，将数据量翻倍，同时提升对左利手用户的识别能力。

### 5.4 解耦架构——CSV 作为中间层

将特征提取（脚本 1）与模型训练（脚本 2）分离，使得：
- 可以多次训练不同架构的模型，而无需重复运行 MediaPipe。
- CSV 文件可以用于其他机器学习框架（sklearn、XGBoost 等）进行对比实验。

### 5.5 生产就绪的工程习惯

- **早停 + 最佳模型保存**：避免过拟合，保留最优状态。
- **学习率自适应调度**：ReduceLROnPlateau 在验证准确率停止提升时自动降低学习率。
- **梯度累积**：模拟更大的 batch size，不增加显存占用。
- **多线程推断**：实时应用中，推断在独立线程运行，不阻塞摄像头采集主循环。

## 总结

本项目展示了一套精巧的 AI 工程流水线：

```
原始图片 → MediaPipe 关键点提取 → CSV → PyTorch MLP 训练 → 实时摄像头推断
```

其核心思想是：**用结构化先验（手部骨骼关键点）替代端到端的像素学习**，在极低的计算成本下实现高精度的手势识别。这是工业界常用的"轻量化 AI"设计哲学的典型体现。
